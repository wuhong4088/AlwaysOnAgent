# 語音使用者流程 (Voice User Flow)

此文件模擬 CookSmart 從「純 UI 介面」升級為「Voice-First Always-On Agent」後的使用者情境對話。

## 情境：下班後打開冰箱不知道煮什麼

**[場景設定]**
使用者下班回家，打開冰箱發現裡面只剩下幾樣東西。雙手正拿著食材，直接對著手機說話。

**使用者 (Voice Input)**: 
「嗨，CookSmart。我冰箱現在有半顆高麗菜、兩顆雞蛋還有一點豬肉。我今晚想吃快一點的，有什麼建議？」

**Agent (Speechmatics 辨識 -> Rasa 處理 -> Rime 語音回覆)**:
*(Agent 在背景自動呼叫 `get_user_dietary_profile`，得知使用者對花生過敏，且最近想吃清淡)*
「沒問題！考慮到你想吃快一點，而且最近比較偏好清淡飲食，我建議可以做『高麗菜豬肉蛋炒』。只需要 15 分鐘喔。不過食譜需要一點青蔥來提味，你冰箱有蔥嗎？」

**使用者 (Voice Input)**:
「沒有蔥耶，那可以幫我把蔥加到明天的購物清單嗎？然後今晚就照你的建議煮吧。」

**Agent (呼叫 MCP Tool `add_to_shopping_list` -> Rime 語音回覆)**:
*(背景成功呼叫 Backend Mock API)*
「好，我已經幫你把青蔥加到 CookSmart 的購物清單了！『高麗菜豬肉蛋炒』的步驟我已經發送到你的螢幕上，你可以隨時一邊看一邊煮。需要我用語音一步一步唸給你聽嗎？」

**使用者 (Voice Input)**:
「不用，我看螢幕就好，謝謝！」

---

## 架構效益
在這個流程中，Agent 展現了：
1. **耳朵與嘴巴**：能聽得懂連續語意並用語音自然回應。
2. **長期記憶大腦**：記得使用者的飲食習慣。
3. **主動執行手腳**：透過 MCP 主動更新購物清單，不再只是一個會說話的文字產生器。
